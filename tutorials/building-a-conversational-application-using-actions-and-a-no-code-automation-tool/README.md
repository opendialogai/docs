# Building a Conversational Application Using Actions and A No-Code Automation Tool

{% embed url="https://youtu.be/CUPLtcBsaV0" %}

## Overview&#x20;

In this tutorial we take a look at how to build a conversational application using OpenDialog’s actions then can connect these actions using webhooks to other no-code tools such as in this case Integromat in order to communicate with Google Sheets.&#x20;

In order for us to get a good grasp of what we can do with OpenDialog actions and how to use them, we will be creating a conversational application that will be a hypothetical company's employee mental wellbeing bot. It's focus will be on first identifying an employee from a company's employees database using their ID, then asking the employee to rate how they feel and finally save how they feel and congratulate them if they feel good otherwise ask them why they feel bad and save that as an additional note in the company's employees database.

{% hint style="info" %}
Note that we will not be making use of any interpreters in this tutorial.
{% endhint %}

You will learn how to:&#x20;

1. Set up actions in OpenDialog.
2. Send and receive data using webhooks.
3. Use attributes in forms.
4. Set up conditions using attributes in intents.
5. Building webhooks in a no-code tool.
6. Simulating reading and writing to and from a database using our webhooks.
